\section{\IfLanguageName{dutch}{Wat is CA Gen?}{What is CA Gen?}}%
\label{sec:Wat is CA Gen?}
 CA Gen is een Computer Aided Software Engineering (CASE)-tool en een vierde generatie ontwikkelomgeving (4GL), oorspronkelijk ontwikkeld in 1987. CA Gen is doorheen de jaren onder verschillende namen bekend geweest, waaronder IEF (Information Engineering Facility), Composer by IEF, Composer, COOL:Gen, Advantage:Gen en AllFusion Gen. Met deze CASE-tool kunnen ontwikkelaars op modelgedreven wijze applicaties ontwerpen via grafische modellering en gestructureerde specificaties, zonder rechtstreeks traditionele programmeersyntax te schrijven. Op basis van deze modellen genereert CA Gen automatisch broncode in verschillende programmeertalen, zoals C en COBOL, en voor diverse platformen. CA Gen ondersteunt onder meer component-based software engineering en verschillende architectuurvormen, waaronder client-server-, batch-, online- en webapplicaties. De tool combineert functioneel ontwerp, datamodellering, technische implementatie en deployment binnen één geïntegreerde ontwikkelomgeving. Alle modellen en gegenereerde artefacten worden opgeslagen in één of meerdere zogenaamde encyclopedieën op het mainframe. Deze encyclopedieën maken gebruik van een DB2-database voor de opslag van metadata en modelinformatie. Binnen ArcelorMittal bevat de encyclopedie verschillende modellen, waarbij elk model een business- of technisch model representeert. De businessmodellen bevatten de logica en specificaties die nodig zijn om een applicatie te genereren, implementeren en testen. 
 
 \section{\IfLanguageName{dutch}{Host encyclopedie}{Host Encyclopedia}}%
 \label{sec:Host Encyclopedie}
 Een Gen-encyclopedie is een centrale opslagplaats van objecten waarmee meerdere gebruikers tegelijkertijd aan meerdere ontwikkelingsprojecten kunnen werken. CA Gen biedt de volgende twee types encyclopedie aan:
 \begin{itemize}
     \item Een host encyclopedie die draait op z/OS gebruike makende van TSO
     \item Een Client Server Encyclopedie (CSE) that draait op een netwerk gebruik makende van één of meerdere werkstations
 \end{itemize}
 De host encyclopedie is een DB2 database dat informatie opslaat dat de Gen Host componenten creëren op meerdere werkstations. Met deze encyclopedie kan je alle kritieke taken controleren vanuit een centrale locatie. Met deze encyclopedie kan heel veel gaan doen, maar in deze context is het belangrijkste de “Check in”, ofwel het uploaden van een model, en de “Check out” ofwel het downloaden van een model.
 
 \section{\IfLanguageName{dutch}{Modellen}{Models}}%
 \label{sec:Modellen}
 CA Gen werkt dus fundamenteel anders dan klassieke programmeertalen. In plaats van rechtstreeks code te schrijven, bouw je een applicatie op via modellen. Die modellen vormen de kern van hoe applicaties worden ontworpen, gegenereerd en onderhouden. CA Gen-modellen zijn abstracte beschrijvingen van een applicatie waarin zowel de functionele logica, datadefinities als gedrag worden vastgelegd, zonder dat er onmiddellijk programmeercode geschreven wordt. Je kan ze zien als een blauwdruk van een applicatie waaruit automatisch code wordt gegenereerd. Volgende soorten modellen bestaan:
 \begin{itemize}
     \item Business models (functionele modellen). 
     Dit zijn de belangrijkste modellen. Deze gaan beschrijven wat de applicatie gaat doen, welke logica uitgevoerd wordt en hoe data verwerkt zal worden zoals bijvoorbeeld berekeningen. Dit gebeurt in een gestructureerde pseudocode waarover later meer.
     \item Data models (datamodellen).
     Deze modellen definiëren entiteiten (bijvoorbeel een klant), attributen (velden) of relaties tussen entiteiten. Dit is vergelijkbaar met een ER-diagram. Deze modellen bepalen ook hoe data uiteindelijk in DB2 wordt opgeslagen.
     \item Action diagrams
     Een action diagram beschrijft de stap-voor-stap logica, hoe data verwerkt wordt en welke acties uitgevoerd worden. Dit ook in een pseudocode.
 \end{itemize}
 Het werken met deze modellen zorgt voor abstractie, je werkt op een hoger niveau dan code wat ervoor zorgt dat code kan gegenereerd worden in verschillende programmeertalen. Anderzijds is deze gegenereerde code wel heel moeilijk te verstaan zonder de modellen.
 
 \section{\IfLanguageName{dutch}{CASE migratie}{CASE migration}}%
 \label{sec:CASE}
 Zoals voorheen uitgelegd is CA Gen een CASE-tool. Een CASE migratie omvat het verplaatsen van softwaresystemen, ontwikkelomgevingen en de bijhorende data van verouderde (legacy) omgevingen naar een ander platform. Dit is een complex proces dat je toelaat om legacy-applicaties te upgraden, te herontwerpen of te migreren naar modernere technologieën, om de prestaties, onderhoudbaarheid of wendbaarheid te verbeteren. 
 Enkele belangrijke aspecten van een CASE-migratie:
 \begin{itemize}
     \item Migratietypen: Migratie kan bestaan uit het verplaatsen, transformeren of vervangen van infrastructuur of applicaties.
     \item Procesfasen: Een succesvolle migratie volgt doorgaans een gestructureerd proces van planning, continu of stapsgewijze uitvoering en validatie.
     \item Migratie kan de efficiëntie van je infrastructuur/applicaties verhogen en je kosten verlagen. Als je wil is AI-integratie ook mogelijk.
     \item Risicoreductie: Een incrementele aanpak, waarbij componenten in kleine stappen worden gemigreerd, zorgt dat het systeem operationeel blijft, waardoor de verstoring van de bedrijfsvoering tot een minimum wordt beperkt.
 \end{itemize}
 
 Veelvoorkomende migratie-uitdagingen:
 \begin{itemize}
     \item Complexiteit: Systemen die gebasseerd zijn op verouderde 4GL-talen vereisen vaak aanzienlijke herontwikkeling om over te stappen naar moderne architecturen.
     \item Compatibiliteit: Het waarborgen dat oude componenten tijdens de overgang blijven functioneren naast de nieuwe.
     \item Tekort aan vaardigheden: De behoefte aan expertise op het gebied van nieuwe frameworks en architecturen.
 \end{itemize}
